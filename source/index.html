What is Magrathea?

Play-by-post roleplaying (RP) sites are about writing. And, tangentially, they’re about code. RP sites can function just fine with the default code hosts provide, but we want these spaces to feel welcoming and homey, thematic and inspiring, we want them to be places we’re eager to cozy up in, sometimes as often as daily. But we’re writers, first and foremost, and these are writing spaces. How can we get what we seek out of code and also keep it out of the way as much as possible so we can focus on what we’re really here to do?

Magrathea is meant to be one possible answer to that question. It’s a collection of resources, and an offer of help. It’s an effort to make creating, maintaining, and participating in RP sites easier and more accessible. It’s a way to give back to our very unique and special community. And it won’t cost you any money.

What Does Magrathea Offer?

So what does Magrathea offer? Broadly speaking, free, high quality, open source code. But that’s an extremely brief way to say a whole lot, so let’s break it down.

Free And Open Source

The resources provided through Magrathea don't cost you any money. There are some terms you'll need to follow, but Magrathea’s publicly available projects exclusively fall under open source licenses, whose terms are meant to be straightforward and to benefit creators and communities. In fact, the terms of most Magrathea projects are as simple as providing credit. You can get full details for any specific Magrathea project from its associated license file.

What is "open source?"

If you've never heard of "open source" before, GitHub provides a few helpful resources. Their "Open Source Guides" offer a definition of "open source" and the "Choose a License" site contains a high-level summary of popular open source licenses.

High Quality Code

"High quality code" here means code that tries its darnedest to be both approachable and accessible.

What is Approachable Code?

Approachable code is code that is:

  • Readable, favoring clarity over concision by, for example, avoiding acronyms and shorthand
  • Organized in a considered and consistent way that includes clear names and structure
  • Well-documented inline where possible and with supplemental documentation as needed
  • Easily maintained. Someone trying to fix a bug or make an update in the code should be able to understand the ripple effects of doing so
  • Built on modern standards, and so is free of outdated, deprecated, or obsolete ways of doing things
  • Something others can learn from or make use of in ways that jumpstart or simplify their own efforts

What is Accessible Code?

Anybody who wants to write should be able to participate in our community regardless of the ways they need or want to approach our sites. From a coding standpoint, that kind of inclusivity doesn’t just magically happen, it’s something we need to actively think about and support. When we write code, we have the power and responsibility to respect the myriad ways that people choose to or must use technology. That, for a few non-exhaustive examples, means coding for people who:

  • Use screen readers or other assistive devices
  • Use the keyboard to navigate a site
  • Use touchscreen devices
  • Are negatively affected by excess motion
  • Modify their browser's default font size
  • Swap out their browser's default font
  • Zoom in on sites
  • Have difficulty differentiating between colors
  • Have sun shining on and washing out their screen
  • Lower their screen's brightness, or shift its colors
  • Become lost within or between pages
  • Have difficulty interacting with small targets
  • Have slower or lower bandwidth internet connections
  • Use devices whose screens are different sizes or resolutions
  • Want to or prefer to use languages other than English, which may be written in different directions
  • Have accessibility needs or desires that change over time or in specific contexts (which is all of us!)

(For a more eloquent list than this, check out Inclusive Design Principles.)

The code provided through Magrathea likely won't be perfectly accessible always, but we want it to be! We're committed to doing the best we can and learning from our shortcomings (so please do reach out if you see something that could be better!).

Writing code that’s approachable and accessible isn’t always easy, but if we keep it in mind upfront and throughout the process, we can hopefully remove a lot of friction for a lot of folks on a lot of fronts.

Vogsphere

The Claim Code Generator

Vogsphere is a tool that enables you to create a claims form on your site. Instead of editing code, your members can just fill out that form to generate exactly what they need to post to make their claims. No more code typos for you to deal with, no more trying to sift through code for them. Win-win.
